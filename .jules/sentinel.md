## 2024-05-24 - [Newline injection bypass in `re.match` validation]
**Vulnerability:** The package name validation function `is_valid_package_name` used `re.match` with a regular expression ending in `$`. This allowed command injection because `$` matches just before a trailing newline (`\n`). An attacker could supply a string like `"com.example.app\n--malicious-flag"` to bypass validation and inject arguments into the adb shell command.
**Learning:** In Python's `re` module, `re.match` combined with `$` is vulnerable to newline injection. `$` does not mean "absolute end of string" as it does in some other regex engines; it means "end of string OR just before the newline at the end of the string".
**Prevention:** Always use `\Z` instead of `$` in Python regular expressions when validating strings to ensure an absolute end-of-string match and prevent newline-based command injection. Additionally, ensure there is a single source of truth for security-critical validation logic (avoid duplicated methods).
